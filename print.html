ZJJ

Run parallel workstreams without conflicts

Create isolated workspaces for each task. Switch instantly. Land cleanly.

What you can do with ZJJ

üîí

Work on multiple features

Each task gets its own isolated workspace. No more stashing, branching, or context switching pain.

‚ö°

Switch in seconds

Jump between workspaces instantly. Your environment, editor state, and terminal history stay intact.

ü§ñ

Run AI agents in parallel

Coordinate 6-12 coding agents safely. Each agent works in isolation with queue-based coordination.

‚úÖ

Land work cleanly

One command merges your changes to main. Built-in sync keeps you up to date with teammates.

How it works

ZJJ combines three tools into a unified workflow:

ComponentWhat it does
JJ WorkspacesHard isolation for each task
Zellij TabsFast context switching between workspaces
SQLite QueueCoordination, retries, and recovery

The workflow

# 1. Create an isolated workspace
zjj add feature-auth --bead BD-123

# 2. Jump into it
zjj focus feature-auth

# 3. Do your work (main stays untouched)
vim src/auth.rs

# 4. Sync with main anytime
zjj sync

# 5. Land your changes
zjj done --message "Add auth"

Before and after

Without ZJJ:

  • 6 agents race on the same working copy ‚Üí duplicated effort, conflicts
  • You switch branches ‚Üí lose editor state, stash changes, forget context
  • Multiple features in progress ‚Üí hard to track what‚Äôs where

With ZJJ:

  • Each agent gets an isolated workspace ‚Üí no conflicts, clear ownership
  • Switch workspaces in 1 second ‚Üí state preserved, instant context
  • See all work at a glance ‚Üí zjj list shows everything

Installation

From source (recommended)

# Clone the repository
git clone https://github.com/lprior-repo/zjj.git
cd zjj

# Build with Moon
moon run :build

# Install the binary
moon run :install

# Verify
zjj --version

From crates.io

cargo install zjj

Your first workspace

1. Initialize ZJJ

# Inside a JJ repository
zjj init

This creates the .zjj/ directory with the state database.

2. Create a workspace

zjj add feature-auth --bead BD-123

This creates:

  • An isolated JJ workspace
  • A Zellij tab for quick switching
  • Association with bead BD-123 for tracking

3. Jump into it

zjj focus feature-auth

Or check where you are:

zjj whereami
# workspace:feature-auth

4. Do your work

Edit files in your isolated workspace. Main stays untouched.

vim src/auth.rs

5. Keep synced

zjj sync

This rebases your workspace onto the latest main.

6. Complete and land

zjj done --message "Add authentication" --push

User Guide

Everything you need to use ZJJ effectively.

Core Concepts

Session

A named, isolated workspace with optional Zellij tab and bead association.

Session = JJ Workspace + Optional Zellij Tab + Optional Bead

Workspace

A complete isolated copy of your repository where you can work without affecting main.

Bead

A task/issue ID that tracks work through the system (e.g., BD-123, zjj-abc45).

Session Lifecycle

Create

zjj add <name> [--bead <id>]

Creates an isolated workspace. Optional --bead links to a task.

zjj add feature-auth --bead BD-123

List

zjj list [--all] [--verbose]

Shows all sessions. Use --all for completed/failed, --verbose for details.

Focus

zjj focus <name>      # Inside Zellij - switch tabs
zjj attach <name>     # Outside Zellij - enter session

Status

zjj status [<name>] [--watch]

Shows workspace state. --watch updates live.

Remove

zjj remove <name> [--force] [--merge]

Removes session. --merge squashes to main first.

Working in a Session

Sync with Main

zjj sync [<name>] [--all]

Rebases your workspace onto latest main. Do this frequently.

Check Location

zjj whereami
# Returns: main | workspace:<name>

View Changes

zjj diff [<name>] [--stat]

Shows differences between your workspace and main.

View Bookmarks

zjj bookmark list
zjj bookmark create <name> [--push]
zjj bookmark delete <name>

Completing Work

done

zjj done [--message <msg>] [--squash] [--keep-workspace]

Merges your workspace to main.

Undo (if needed)

zjj undo              # Undo last done
zjj undo --list       # Show undo history
zjj revert <name>     # Revert specific session

Queue Coordination

For multi-agent or coordinated workflows.

AI Agent Guide

Complete guide for AI agents using ZJJ. Start here.

7 Mandatory Rules

1. Never use cargo ‚Äî use moon run :check|:test|:build|:quick|:ci
2. Never use unwrap/expect/panic ‚Äî Result<T,E> everywhere
3. Never modify lint config ‚Äî fix code, not lint
4. Never skip git push ‚Äî not done until pushed
5. After br sync ‚Äî git add .beads/ && git commit
6. Load functional-rust-generator skill for all Rust
7. Manual testing mandatory ‚Äî test actual behavior

The 7-Step Workflow

1. ORIENT     ‚Üí zjj whereami
2. REGISTER   ‚Üí zjj agents register
3. ISOLATE    ‚Üí zjj work <name> --bead <id>
4. ENTER      ‚Üí cd $(zjj context --field=workspace.path)
5. IMPLEMENT  ‚Üí moon run :check && moon run :test
6. HEARTBEAT  ‚Üí zjj agents heartbeat -c "building"
7. COMPLETE   ‚Üí zjj done -m "message" && git push

Common Errors

ERR_NOT_JJ_REPO (5)

Message: Not in a JJ repository

Fix:

jj init && zjj init

Or clone existing:

jj git clone https://github.com/user/repo.git
cd repo && zjj init

ERR_ALREADY_EXISTS (3)

Message: Workspace already exists

Fix:

# Remove first
zjj remove existing-name

# Or use idempotent flag
zjj add new-name --idempotent

ERR_NOT_FOUND (4)

Message: Workspace not found

Fix:

zjj list  # Check available names
zjj add missing-name

ERR_LOCK_CONFLICT (7)

Message: Resource is locked

Fix:

zjj status <workspace>  # Check who holds lock
# Wait for lock to expire (default: 1 hour)
# Or force release:
zjj unlock <workspace>

ERR_CONFLICT (8)

Message: Merge conflict detected

Fix:

zjj focus <workspace>
jj resolve  # Resolve each conflict
zjj sync    # Verify resolution

ERR_DATABASE_ERROR (6)

Message: Database operation failed or Database corruption detected

Fix:

zjj doctor --fix
zjj recover --last
cat .zjj/recovery.log

ERR_VALIDATION_ERROR (13)

Message: Invalid input

Rules:

  • Workspace names: lowercase, alphanumeric, dashes only
  • Must start with a letter
  • Max 64 characters
# Valid
zjj add feature-auth
zjj add fix-123

# Invalid
zjj add Feature-Auth    # Uppercase
zjj add feature_auth    # Underscore
zjj add 123-feature     # Starts with number

ERR_ZELLIJ_ERROR (11)

Message: Zellij command failed or Zellij not found

Fix:

# Check installation
zellij --version

# Install if missing
cargo install zellij
# or
brew install zellij

# Or skip Zellij
zjj add <name> --no-zellij

ERR_WORKSPACE_ERROR (10)

Message: Workspace operation failed

Fix:

zjj status <workspace>
jj workspace list

# If corrupted
zjj remove <workspace> --force
zjj add <workspace>

ERR_RECOVERY_NEEDED (14)

Message: Corruption detected, recovery required

Fix:

zjj doctor --fix
zjj recover --diagnose
zjj recover --last

Recovery Procedures

Database Recovery

# Check health
zjj doctor

# Auto-fix
zjj doctor --fix

# If that fails
zjj recover --diagnose
zjj recover --last

# Check logs
cat .zjj/recovery.log

Workspace Recovery

# Validate
zjj integrity validate <workspace>

# Repair
zjj integrity repair <workspace> --force

# Restore from backup
zjj integrity backup list
zjj integrity backup restore <id>

Session Recovery

# List checkpoints
zjj checkpoint list

# Restore checkpoint
zjj checkpoint restore <id>

Diagnostics

Health Check

zjj doctor --verbose

System State

zjj context --json
zjj introspect --ai --json

Event Log

zjj events --follow
zjj events --session <name>

Query State

zjj query session-exists <name>
zjj query session-count
zjj query can-run

Clean Up

Stale Sessions

# Preview
zjj clean --dry-run

# Remove
zjj clean --force

Invalid Sessions

# Preview
zjj prune-invalid --dry-run

# Remove all
zjj prune-invalid --yes

Orphaned Workspaces

jj workspace list
jj workspace forget <name>
zjj clean --force
